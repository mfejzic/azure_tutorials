Launch type: Non-serverless | Infrastructure size: Large | Security: High

  • Virtual Private Cloud: VPC is created in the US-East-1 and US-West-2 region.
  • Network Access Control List: NACL is created and associated with each VPC, allowing or denying traffic in and out of subnets within the VPC.
  • Subnets and Gateways: Multiple public and private subnets are created across different Availability Zones in the region. An internet gateway and elastic IPs are created for network address translation gateways and placed in public subnets to provide external network access to private resources.
  • Security Groups: Security groups are defined for different purposes, such as allowing SSH access to a bastion host, allowing HTTP/HTTPS traffic for a load balancer, and controlling traffic to private instances.
  • Key Pairs and Instances: Key pairs are created for admin to SSH into EC2 instances. Amazon machine images are used to launch EC2 instances in public and private subnets automatically using data block.
  • Application Load Balancer and Auto Scaling: An Application Load Balancer is created to distribute incoming traffic to EC2 instances. Auto Scaling Groups are used to automatically adjust the number of EC2 instances based on demand.
  • Amazon S3 and Identity and Access Management (IAM): An Amazon S3 bucket is created, and objects are uploaded, with encryption and versioning enabled. IAM roles and policies are established to allow EC2 instances to access the S3 bucket securely and fetch static files.
  • CloudWatch and Simple Notification Service: CloudWatch alarms are set up to monitor S3 metrics, and alerts are sent through an SNS topic.
  • Route 53: I created route 53 to configure DNS records which implement failover policies and health checks for highly available access to resources in different regions.

Launch type: Serverless | Infrastructure size: Small | Security: Low

  • S3 Bucket: Creates an S3 bucket to store static website files. Enabled with server-side encryption for all objects using the AES256 algorithm and versioning to protect against accidental object deletion.
  • ACM Certificate: Uses data block to retrieve an ACM certificate for the domain, ensuring the most recent certificate with the type "AMAZON_ISSUED."
  • AWS CloudFront: Creates a CloudFront distribution to serve static web pages. Configures a TLS certificate from ACM for enhanced security.
  • Route 53: Uses data block to retrieve an existing Route 53 hosted zone for the domain. Creates primary and secondary DNS records for routing traffic to the CloudFront distribution. Implements health checks for the primary and secondary records to monitor their availability.

Tower of Hanoi - Java Project

  • User Input Validation: It continuously accepts user input for selecting the starting and target towers.The code validates user input to ensure they select valid tower options and that the target tower is different from the starting tower.
  • Tower Movement and Game Logic: It checks if the user can move a disk from the starting tower to the target tower based on the game rules. If a move is valid, it updates the tower configuration and counts the number of moves and provides feedback whether successful or not.
  • Winning Condition: The code checks if the user has successfully completed the Tower of Hanoi puzzle by moving all disks to another tower. If the win condition is met, it displays the total number of moves and ends the game.
  • Display of Tower State: The code continuously displays the current state of the three towers, showing the arrangement of disks.
  • Termination: The game continues to run until the user successfully completes the puzzle, after which it terminates.
